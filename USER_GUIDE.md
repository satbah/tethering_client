# Ticker User Guide Draft

## 1. この機器でできること

この Ticker は、ESP32-S3 を使ってスマホのテザリングへ接続し、為替 API から USD/JPY を取得して画面に表示する試作機です。

通常は以下の流れで動きます。

1. 起動する
2. 保存済みのテザリング設定で Wi-Fi 接続する
3. API へアクセスする
4. 取得結果を表示する
5. 一定時間待ってから再度起動する

初期設定や設定変更が必要なときは、通常動作ではなく AP mode を使います。

## 2. 使う前に必要なもの

- Ticker 本体
- USB 給電
- スマートフォン
- スマホ側のインターネット共有機能
- テザリングの SSID とパスワード

事前に確認しておく項目:

- スマホのテザリング名
- スマホのテザリング用パスワード
- 利用する API URL

## 3. 2つの動作モード

### Ticker mode

通常運用モードです。保存済み設定でテザリングへ接続し、レート取得を試みます。

### AP mode

設定モードです。Ticker 自身が Wi-Fi アクセスポイントになり、スマホのブラウザから設定します。

AP mode の用途:

- 初回セットアップ
- テザリング SSID / パスワード変更
- API URL 変更
- 取得周期変更

## 4. 初回セットアップ

### 4.1 AP mode に入る

1. 本体を起動します。
2. BOOT ボタンを約 3 秒長押しします。
3. 画面にセットアップ用 AP の案内と QR が表示されます。

想定動作:

- SSID は `TickerPoc`
- 設定ページの URL は `http://192.168.4.1/`

### 4.2 スマホを接続する

1. スマホの Wi-Fi 設定を開きます。
2. `TickerPoc` に接続します。
3. 画面の QR を読むか、ブラウザで `http://192.168.4.1/` を開きます。

うまく遷移しない場合は、モバイル通信のままではなく、`TickerPoc` に接続できているかを先に確認してください。

### 4.3 設定ページで入力する

設定ページには次の項目があります。

- `STA SSID`: スマホのテザリング SSID
- `STA Password`: スマホのテザリングパスワード
- `API URL`: データ取得先 URL
- `Poll interval (sec)`: 取得周期
- `通信状態を音で知らせる`: 通知音の ON/OFF（初期値は ON）

入力後の操作:

- `保存`: 設定だけ保存します。本体は AP mode のままです。
- `保存して再起動`: 設定を保存して通常モードへ戻します。

音通知設定の目安:

- ON: 接続開始・成功・失敗の音で状態を把握しやすい
- OFF: 静かな環境で運用したい場合に有効

推奨手順:

1. 必要項目を入力する
2. `保存して再起動` を押す
3. スマホの設定画面でインターネット共有のページを開く
4. 本体が通常起動して接続を試すのを待つ

## 5. 通常運用

設定後は、短押し起動や通常起動では Ticker mode で動きます。

通常運用時の考え方:

- 本体は保存済みの SSID とパスワードで接続を試みる
- 一度 API 取得を行う
- その後は設定された周期で再試行する

スマホ側の注意点:

- テザリングを有効にしておく
- 可能ならインターネット共有の設定画面を開いたままにする
- 省電力設定でテザリングが止まらないようにする

機種によっては、画面ロックや省電力制御でテザリングの見え方や再接続性が変わります。

## 6. 画面の見方

この試作では、画面に接続状態や取得結果のログが表示されます。

主な見どころ:

- Wi-Fi 接続に成功したか
- NTP 同期が進んだか
- HTTP アクセスが成功したか
- 最後に取得した USD/JPY がいくつか

AP mode 中は、セットアップ用 SSID や QR 表示が優先されます。

通常モードの注意:

- BOOT短押しで設定ページには入りません
- 設定変更は AP mode（BOOT長押し）から行ってください

## 7. 設定を変えたいとき

以下のときは AP mode に入り直してください。

- スマホを買い替えた
- テザリング名を変えた
- パスワードを変えた
- API URL を変えた
- 更新間隔を変えたい

手順は初回セットアップと同じです。

1. BOOT 長押しで AP mode に入る
2. `TickerPoc` に接続する
3. 設定ページを開く
4. 変更して `保存して再起動` を押す

## 8. よくあるトラブル

### `TickerPoc` が見えない

- BOOT 長押し時間が短い可能性があります
- 本体を再起動して、約 3 秒を目安に押し続けてください

### 設定ページが開けない

- スマホが `TickerPoc` へ接続されているか確認してください
- URL を直接 `http://192.168.4.1/` と入力してください

### 保存したのに動かない

- `保存` だけ押した場合は AP mode のままです
- `保存して再起動` を押してください

### 再起動後につながらない

- スマホのインターネット共有が有効か確認してください
- SSID とパスワードの入力ミスを疑ってください
- スマホの設定画面でインターネット共有ページを開いて再試行してください

### しばらくするとつながらない

- スマホ側の自動オフや省電力制御の可能性があります
- 画面ロック時のテザリング維持設定を確認してください

## 10. 開発者向けメモ

エンドユーザー向けではありませんが、開発中の確認に使う要点を残します。

- AP mode SSID: `TickerPoc`
- AP config URL: `http://192.168.4.1/`
- BOOT 長押し: 約 3 秒
- `保存` は再起動しない
- `保存して再起動` は AP 切断を伴う
